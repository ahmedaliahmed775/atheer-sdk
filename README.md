# Atheer SDK - مكتبة مدفوعات NFC الذكية لـ Android

<div dir="rtl">

## 🌟 نظرة عامة

**Atheer SDK** هي مكتبة Android متطورة بدون واجهة (Headless)، مصممة للتكامل مع تطبيقات المحافظ الرقمية (Host Wallet Apps). تمكن المكتبة هذه التطبيقات من تنفيذ مدفوعات NFC اللاتلامسية بأمان فائق، مع دعم كامل للوضع غير المتصل بالإنترنت (Offline-First) وتوجيه صارم للبيانات عبر الشبكة الخلوية لضمان الامتثال للمعايير البنكية.

> **ملاحظة:** المكتبة تركز على الجوانب التقنية والأمنية لعمليات الدفع والـ Tokenization، بينما تظل إدارة حسابات المستخدمين والواجهات من مسؤولية التطبيق المضيف.

---

## 🏗️ المعمارية التقنية

تعتمد المكتبة معمارية **Clean Architecture** لضمان الفصل بين المسؤوليات وسهولة الاختبار:

- **الواجهة (Facade):** `AtheerSdk.kt` - نقطة التعامل الوحيدة والمركزية.
- **الأمان (Security):** 
    - `AtheerKeystoreManager`: إدارة مفاتيح AES-256-GCM المحمية عتادياً.
    - `AtheerTokenManager`: إدارة خزنة الرموز المميزة (Tokens) لمرة واحدة.
- **الاتصال (NFC/HCE):** 
    - `AtheerApduService`: محاكاة البطاقة وتطبيق الجدار الناري للأمان.
    - `AtheerNfcReader`: تحويل الهاتف إلى جهاز نقطة بيع (SoftPOS).
- **البيانات (Data):** 
    - `AtheerDatabase`: قاعدة بيانات Room مشفرة لتخزين العمليات "أوفلاين".
    - `AtheerNetworkRouter`: توجيه خلوي إجباري لطلبات الدفع الحساسة.

---

## 🛡️ الجدار الناري للأمان (Security Firewall)

تنفرد المكتبة بنظام حماية ديناميكي يتحكم في العمليات قبل وصولها للتوكن:
1. **الحد المالي (Offline Limit):** يمكن للمستخدم تحديد سقف مالي لكل عملية دفع (مثلاً: لا تسمح بأي عملية فوق 100 ريال).
2. **التحقق اللحظي:** عند اقتراب الهاتف من جهاز التاجر، يقوم الـ SDK باستخراج المبلغ المطلوب ومقارنته بالحد المسموح.
3. **الرفض البنكي:** في حال تجاوز الحد، يتم الرفض فوراً بكود `6A80` وإرسال تنبيه للتطبيق المضيف عبر `Broadcast`.
4. **الاستخدام لمرة واحدة:** يتم مسح الحد والتوكن فور نجاح العملية لضمان عدم التكرار.

## 🚀 دليل الاستخدام السريع

### 1. التهيئة (Initialization)
يجب استدعاء التهيئة مرة واحدة، ويفضل أن يكون ذلك في كلاس `Application`:

```kotlin
AtheerSdk.init(
    context = this,
    merchantId = "YOUR_MERCHANT_ID",
    apiBaseUrl = "https://api.atheer.com"
)
```

### 2. ضبط حدود الأمان
قبل تفعيل الدفع، يمكن للتطبيق المضيف ضبط الحد المسموح به:

```kotlin
val sdk = AtheerSdk.getInstance()
sdk.setNextOfflineLimit(500) // السماح بعملية واحدة بحد أقصى 500 ريال
```

### 3. جلب وتزويد الرموز (Provisioning)
للدفع في وضع الأوفلاين، يجب شحن الخزنة بالرموز المميزة:

```kotlin
// جلب 10 رموز من السيرفر بحد دفع 1000 ريال لكل منها
sdk.fetchAndProvisionTokens(authToken, count = 10, limit = 1000)
```

### 4. استقبال تنبيهات الرفض
عندما يرفض الـ SDK عملية بسبب تجاوز الحد، يرسل Broadcast Action:
`com.atheer.sdk.ACTION_PAYMENT_REJECTED`

## 🛠️ العمليات الأساسية

| الوظيفة | الدالة | الوصف |
| :--- | :--- | :--- |
| **دفع مباشر** | `processTransaction()` | معالجة الدفع وحفظه محلياً أو إرساله فوراً. |
| **شحن (Charge)** | `charge()` | إرسال التوكن الملتقط من POS إلى السيرفر لإتمام الشحن. |
| **مزامنة** | `syncPendingTransactions()` | رفع جميع العمليات المخزنة محلياً عند توفر الإنترنت. |
| **قراءة NFC** | `AtheerNfcReader` | تستخدم في وضع SoftPOS لاستقبال الدفع من بطاقات أخرى. |

## 📦 بناء المشروع والدمج

### بناء ملف AAR
للحصول على نسخة المكتبة القابلة للدمج:
```bash
./gradlew :atheer-sdk:assembleRelease
```
المسار: `atheer-sdk/build/outputs/aar/atheer-sdk-release.aar`

### المتطلبات
- **Android SDK**: API 24+
- **NFC Hardware**: مطلوب لعمليات اللمس.
- **التصاريح**: (NFC, INTERNET, ACCESS_NETWORK_STATE, CHANGE_NETWORK_STATE).

---

## 🔐 معايير الأمان
- **Android Keystore**: لا تخرج المفاتيح أبداً من النطاق الآمن للجهاز.
- **Nonce Generation**: رقم عشوائي فريد لكل معاملة لمنع هجمات Replay.
- **Cellular Routing**: حماية البيانات من الاعتراض عبر شبكات الواي فاي العامة.
- **Payload Encryption**: تشفير كامل لجسم الطلب (JSON) باستخدام AES-GCM.
